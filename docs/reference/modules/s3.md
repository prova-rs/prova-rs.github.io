# s3

A thin object-storage client — enough to assert on an S3 dependency: an object the app wrote, its contents, a listing. Uses **path-style addressing** (MinIO-compatible) and rustls, so real HTTPS S3 endpoints work too. `s3.container` is the ephemeral **MinIO** recipe, the object-storage counterpart to [`db.postgres`](db.md#recipes-dbpostgres--dbmysql).

## `s3.connect(opts)`

```lua
s3.connect(opts) --> S3Bucket
```

Connects to one bucket on an S3/MinIO endpoint.

| Option | Type | Description |
|---|---|---|
| `endpoint` | `string` | Required. E.g. `"http://127.0.0.1:9000"` |
| `bucket` | `string` | Required. The bucket name |
| `access_key` | `string?` | Access key |
| `secret_key` | `string?` | Secret key |
| `region` | `string?` | Default `"us-east-1"` |
| `create` | `boolean?` | Create the bucket (idempotent — "already exists" is tolerated). Because it hits the network, it doubles as a readiness probe |

**Returns:** an `S3Bucket`. Async — call inside a fixture factory or test body.

```lua
local bucket = s3.connect{
  endpoint = "http://127.0.0.1:9000",
  bucket = "uploads",
  access_key = "minioadmin",
  secret_key = "minioadmin",
  create = true,
}
```

## `S3Bucket`

All methods are async.

| Member | Type | Description |
|---|---|---|
| `b:put(key, data)` | | Write an object (the bytes of the string) |
| `b:get(key)` | `→ string` | Read an object's contents (raises if it does not exist) |
| `b:exists(key)` | `→ boolean` | Whether an object exists (via HEAD) |
| `b:delete(key)` | | Delete an object |
| `b:list(prefix?)` | `→ string[]` | The object keys under an optional prefix |
| `b:close()` | | No-op; present for `ctx:manage` symmetry |

```lua
local store = prova.fixture("s3", Scope.File, function(ctx)
  return s3.container(ctx).bucket
end)

prova.group("s3", { requires = { "docker" } }, function(g)
  g:test("put / get round-trips an object", function(t)
    local b = t:use(store)
    b:put("greeting.txt", "hello world")
    t:expect(b:get("greeting.txt")):equals("hello world")
  end)

  g:test("exists, list, and delete", function(t)
    local b = t:use(store)
    b:put("a/one.txt", "1")
    b:put("a/two.txt", "2")
    t:expect(b:exists("a/one.txt")):is_true()
    local keys = b:list("a/")
    t:expect(#keys):equals(2)
    b:delete("a/one.txt")
    t:expect(b:exists("a/one.txt")):is_false()
  end)
end)
```

## `s3.container(ctx, opts)`

```lua
s3.container(ctx, opts?) --> S3Resource
```

Provisions an ephemeral **MinIO** (image `minio/minio`, command `server /data`), creates a bucket (which doubles as the readiness gate), and returns a bucket client — all managed by the scope. Requires the [`docker`](docker.md) module at call time — gate with `requires = { "docker" }`.

| Parameter | Type | Description |
|---|---|---|
| `ctx` | `Context` | The fixture/test context (first argument, required) |
| `opts.image` | `string?` | Full image ref; overrides `tag` |
| `opts.tag` | `string?` | Image tag (default `"latest"`) |
| `opts.bucket` | `string?` | Bucket to create (default `"prova"`) |
| `opts.access_key` | `string?` | Default `"minioadmin"` |
| `opts.secret_key` | `string?` | Default `"minioadmin"` |
| `opts.timeout` | `string?` | Readiness deadline (default `"60s"`) |

**Returns:** an `S3Resource`:

| Member | Type | Description |
|---|---|---|
| `endpoint` | `string` | `http://127.0.0.1:<host_port>` |
| `bucket` | `S3Bucket` | A client bound to the created bucket (managed) |
| `container` | `Container` | The managed [container handle](docker.md#container) |
| `access_key` | `string` | The credentials in effect — handy for pointing the app under test at the store |
| `secret_key` | `string` | |
